# 🍽️ Swiggy Sales Data Analysis (SQL Project)

## 📌 Project Overview
This project focuses on analyzing Swiggy food delivery data using SQL to extract meaningful business insights. The project includes data cleaning, data modeling using a star schema, and advanced analytical queries.

---

## 🎯 Objectives
- Perform data cleaning and validation
- Design a star schema database
- Analyze sales performance and trends
- Generate business insights for decision-making

---

## 🛠 Tools Used
- SQL (SQL Server Management Studio / SQL Server)
- Excel (Dataset)

---

## 🧱 Data Modeling (Star Schema)
The project uses a star schema with:

### 🔹 Fact Table:
- `fact_swiggy_orders`

### 🔹 Dimension Tables:
- `dim_date`
- `dim_location`
- `dim_restaurant`
- `dim_category`
- `dim_dish`

---

## 🧹 Data Cleaning
- Checked for NULL values
- Identified blank records
- Removed duplicate rows

---

## 📊 Key KPIs
- Total Orders
- Total Revenue
- Average Price
- Average Rating

---

## 📈 Business Analysis

### 📅 Time-Based Analysis
- Monthly order trends
- Quarterly trends
- Yearly trends
- Orders by day of week

### 🌍 Location Analysis
- Top cities by orders and revenue
- State-wise revenue contribution

### 🍴 Product & Category Analysis
- Top restaurants by revenue
- Most ordered dishes
- Category performance
- Best-rated categories

### 💰 Price Analysis
- Orders by price range

### ⭐ Rating Analysis
- Rating distribution
- Most frequent ratings

---

## 🔍 Key Insights
- Few cities contribute majority of revenue
- Certain categories have high orders but lower ratings
- Most orders fall within mid-price range
- Top restaurants generate significant revenue share
